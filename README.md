# Deepfake Audio Detection

## Project Overview

The field of intelligence is advancing rapidly and this has made it possible to generate very realistic speech also known as deepfake audio. While deepfake audio has some uses, such as virtual assistants and voice cloning for people who need it it also poses some serious threats like identity theft and financial fraud.

The goal of this project is to create a learning system that can tell the difference between real human speech and deepfake audio recordings. The system takes an audio recording as input. Says whether the speech is Genuine which means it is from a human or Deepfake which means it is generated by a computer.

We used a end to end approach that involves converting audio signals into Mel spectrogram representations and then using a Convolutional Neural Network to classify them. The final model worked well on test samples that it had not seen before and we also made a Streamlit web application that allows users to upload audio files and get predictions along with confidence scores.

# Motivation

Human speech has acoustic structures that are created naturally by the vocal tract. Computer systems that generate speech try to copy these characteristics. They often leave behind subtle artifacts.

Deepfake audio may have some issues like:

* Missing harmonics

* Irregular spectral patterns

* energy distributions

* Temporal inconsistencies

* frequency relationships

These issues are hard to see directly in the raw audio waveforms but they become more visible when we transform the audio into time-frequency representations.

---

# Why Mel Spectrograms?

Audio signals are originally represented as one-dimensional waveforms. Deep learning models like Convolutional Neural Networks work well on image-like inputs.

Mel spectrograms convert audio into two- representations where:

* The x-axis shows time

* The y-axis shows frequency, which is on the Mel scale

* The color intensity shows the signal energy

The Mel scale is similar to how humans perceive sound, which makes it very effective for tasks that involve analyzing speech.

---

# Why Convolutional Neural Networks?

Convolutional Neural Networks are designed to learn spatial patterns from images.

Since Mel spectrograms are similar to images Convolutional Neural Networks can identify acoustic patterns associated with genuine and deepfake speech.

The advantages of Convolutional Neural Networks include:

* They can automatically extract features

* They reduce the need for features

* They are robust to variations

* They have classification accuracy

For these reasons Convolutional Neural Networks are one of the most practical approaches for classifying audio spectrograms.

# Methodology

## Step 1: Audio Preprocessing

Each audio file goes through preprocessing stages.

### Audio Loading

We load the recordings using the Librosa library.

```python

librosa.load()

```

---

### Resampling

We resample all the recordings to 16 kHz. This makes sure that the sampling frequency is the same across the dataset.

---

### Mel Spectrogram Generation

We generate Mel spectrograms using the librosa.feature.melspectrogram function.

We use the following parameters:

* Sampling Rate: 16 kHz

* Number of Mel Bands: 128

---

### Decibel Conversion

We convert the power spectrogram to the scale using the librosa.power_to_db function. This makes it easier to see the differences.

---

### Fixed-Length Processing

Since the audio durations are different we. Truncate the spectrograms to make sure they are all the same size.

The target spectrogram dimensions are 128 × 100. This allows us to process the audio in batches within the Convolutional Neural Network.

---

# Deep Learning Architecture

The classification model is a Convolutional Neural Network.

The architecture includes:

* Convolutional Layers

* ReLU Activation Functions

* Pooling Layers

* Connected Layers

* Softmax Output Layer

The network learns to identify the differences between genuine and deepfake speech directly from the Mel spectrogram inputs.
